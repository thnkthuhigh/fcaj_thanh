During the internship, I participated in **one main technology event** within the FCAJ program on **May 23, 2026** at the AWS office in Bitexco. Even though the number of events I attended was limited, this session was still meaningful because it helped me view CloudDoc from the perspective of system design, data quality, and long-term backend thinking rather than only immediate implementation tasks.

### Event attended

- [4.1 - AWS Vietnam Community Day in May 2026: perspectives on AI, data, and system architecture](4.1-event1/)

### Key value gained

The biggest takeaway for me was learning to place technology decisions inside real operating context. A backend feature may work correctly in code, but it is still incomplete if the surrounding data model, observability, scaling path, and cost implications are not considered. That mindset influenced the way I later approached CloudDoc.
