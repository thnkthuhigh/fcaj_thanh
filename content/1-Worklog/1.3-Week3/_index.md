### Week 3 Objectives:

* Understand the basic AWS network structure with VPC, subnets, route tables, and Internet Gateway.
* Learn why public and private subnets matter when deploying backend services and databases.
* Strengthen security thinking at both the network and access-control layers.

### Tasks to be carried out this week:
| Day | Task | Start Date | Completion Date | Reference Material |
| --- | --- | --- | --- | --- |
| 1 | Study VPC architecture, CIDR ranges, subnets, and the reason AWS networks are segmented this way. | 01/05/2026 | 01/05/2026 |  |
| 2 | Practice creating route tables, Internet Gateway connections, and basic outbound traffic paths. | 04/05/2026 | 04/05/2026 |  |
| 3 | Compare public and private subnets in the context of backend and database deployment. | 05/05/2026 | 05/05/2026 |  |
| 4 | Review Security Groups, NACLs, and different layers of access protection. | 06/05/2026 | 06/05/2026 |  |
| 5 | Bring the concepts together to understand a multi-layer architecture for the CloudDoc system. | 07/05/2026 | 07/05/2026 |  |


### Week 3 Achievements:

* Understood how to organize network layers for a backend system on AWS.
* Learned why network separation matters when backend services, databases, and storage are deployed together.
* Built a stronger foundation for reading and designing the CloudDoc architecture later on.
