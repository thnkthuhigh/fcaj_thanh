**This page** summarizes my 12-week internship journey in the First Cloud AI Journey program in the role of a **Backend Developer** for the CloudDoc project. Each week follows the same structure: `Week Objectives`, `Tasks to be carried out this week`, and `Week Achievements`, so the worklog remains aligned with the expected internship-report format.

The timeline shows how I progressed from AWS foundation labs to backend design and implementation for CloudDoc using Node.js, Express, PostgreSQL, and Amazon S3. The internship timeline is organized as follows:

**Week 1:** [Getting familiar with the FCAJ program, setting up an AWS account, and completing foundation labs](1.1-week1/)

**Week 2:** [Practicing core AWS services: EC2, S3, and IAM](1.2-week2/)

**Week 3:** [Deploying AWS network infrastructure and strengthening access security](1.3-week3/)

**Week 4:** [Learning at the AWS office and kicking off the CloudDoc project](1.4-week4/)

**Week 5:** [Researching backend direction and analyzing the CloudDoc problem space](1.5-week5/)

**Week 6:** [Participating in technology events and expanding backend system thinking](1.6-week6/)

**Week 7:** [Self-study, backend architecture design, and development environment preparation](1.7-week7/)

**Week 8:** [Bootstrapping the CloudDoc backend and designing the data model](1.8-week8/)

**Week 9:** [Building core document APIs and PostgreSQL data queries](1.9-week9/)

**Week 10:** [Integrating S3 uploads with presigned URLs and completing document business logic](1.10-week10/)

**Week 11:** [Strengthening backend security, logging, and operability](1.11-week11/)

**Week 12:** [Supporting frontend integration, end-to-end testing, and final reporting](1.12-week12/)
